Mobile App Sessions (Refresh Tokens)

Every mobile login creates one refresh-token record so the app can silently renew its session without asking a parent to log in every day. Rows are rotated (replaced) on each renewal, and revoked on logout. This page shows the current counts and lets you run the cleanup job on demand instead of waiting for its automatic daily run.

A row is only deleted once it has been expired or revoked for at least 7 day(s) — this grace period keeps recent history around in case a session needs to be investigated.
